# Topical Authority (Research-Based)

## Research Basis
- **Backlinko 2026**: "The more comprehensively you cover a topic, the more likely you are to show up everywhere"
- **HubSpot**: topic cluster model — pillar + cluster posts drive 2x more traffic
- **Ahrefs**: sites ranking for 1 topic keyword tend to rank for related keywords
- **Google**: topical depth = authority signal (E-E-A-T)

## What Is Topical Authority

### Definition
Google sees your site as THE authority on a topic when you cover it comprehensively — not just one post, but an entire ecosystem of interconnected content.

### Why It Matters (2026)
- **Traditional SEO**: rank for individual keywords
- **Topical Authority**: rank for entire topic clusters
- **AI Search**: AI cites comprehensive sources
- **Competitive moat**: hard to replicate deep coverage

## Topic Cluster Architecture

### HubSpot Model
```
                    [Pillar Page]
                    (2,500-5,000 words)
                    Broad topic overview
                         │
        ┌────────────────┼────────────────┐
        │                │                │
   [Cluster 1]      [Cluster 2]      [Cluster 3]
   (1,000-2,000)    (1,000-2,000)    (1,000-2,000)
   Specific subtopic  Specific subtopic  Specific subtopic
        │                │                │
   ┌────┤           ┌────┤           ┌────┤
   │    │           │    │           │    │
  [A]  [B]         [C]  [D]         [E]  [F]
  Sub   Sub         Sub   Sub         Sub   Sub
```

### Rules
1. **Pillar links to all clusters** — one-way from pillar to cluster
2. **All clusters link back to pillar** — one-way from cluster to pillar
3. **Clusters link to related clusters** — contextual cross-linking
4. **No duplicate topics** — each cluster covers unique subtopic
5. **Consistent naming** — URL structure reflects hierarchy

### URL Structure
```
/pillar-topic/                    ← Pillar page
/pillar-topic/subtopic-1/         ← Cluster post
/pillar-topic/subtopic-2/         ← Cluster post
/pillar-topic/subtopic-3/         ← Cluster post
```

## Building Topical Authority

### Step 1: Topic Research
1. **Seed keyword**: broad topic (e.g., "SEO")
2. **Keyword research**: find all related keywords
3. **SERP analysis**: what topics does Google show for each?
4. **Cluster by SERP similarity**: group keywords with same results
5. **Validate**: search top 3 keywords per cluster — same results?

### Step 2: Architecture Design
```
Topic: SEO
  Pillar: "Panduan SEO Lengkap 2026" (3,000 words)
  
  Cluster 1: "Keyword Research"
    - Cara riset keyword untuk pemula
    - Tool riset keyword gratis
    - Search intent classification
    - Long-tail keyword strategy
  
  Cluster 2: "On-Page SEO"
    - Optimasi title tag dan meta description
    - Internal linking strategy
    - Content structure untuk SEO
    - Image optimization untuk SEO
  
  Cluster 3: "Technical SEO"
    - Core Web Vitals optimization
    - Sitemap dan robots.txt
    - Mobile-first indexing
    - Schema markup guide
  
  Cluster 4: "Link Building"
    - Guest posting strategy
    - Broken link building
    - HARO link building
    - Content-driven links
  
  Cluster 5: "SEO Analytics"
    - Google Search Console setup
    - GA4 for SEO
    - Rank tracking
    - SEO reporting
```

### Step 3: Content Creation Order
1. **Pillar first** — establish the hub
2. **Cluster 1-2** — foundational subtopics
3. **Cross-links** — connect all clusters
4. **Cluster 3-5** — expand coverage
5. **Data-driven content** — original research per cluster
6. **Refresh cycle** — quarterly updates

### Step 4: Internal Linking Strategy
```
Pillar → Cluster: "Learn more about [subtopic]" (anchor text = subtopic keyword)
Cluster → Pillar: "For a complete guide, see our [pillar]" (anchor text = pillar keyword)
Cluster → Cluster: "Related: [other cluster topic]" (contextual link)
```

### Anchor Text Rules
- **Natural variation**: not all exact match
- **Descriptive**: tells user what they'll find
- **Varied**: mix of exact, partial, and generic
- **Contextual**: placed within relevant content

## Authority Measurement

### How to Measure
1. **Keyword breadth**: ranking for how many keywords in topic?
2. **SERP visibility**: appearing for how many related queries?
3. **Cluster coverage**: how many cluster posts published?
4. **Internal link density**: average internal links per page in topic
5. **Traffic distribution**: is traffic spread across clusters?

### Authority Score Formula
```
Authority Score = (Cluster Posts / Total Possible) × 0.3
                + (Keywords Ranking / Target Keywords) × 0.3
                + (Internal Links / Target Links) × 0.2
                + (Traffic Growth / Previous Period) × 0.2
```

### Benchmarks
| Authority Level | Cluster Posts | Keywords | Internal Links |
|----------------|---------------|----------|----------------|
| Beginner | 5-10 | 50-100 | 10-20 |
| Intermediate | 15-25 | 200-500 | 50-100 |
| Advanced | 30-50 | 500-1000 | 100-200 |
| Authority | 50+ | 1000+ | 200+ |

## AI Search Optimization for Clusters

### Query Fan-Out Coverage
- Each cluster post answers a specific sub-query
- Pillar page provides comprehensive overview
- Together: complete coverage of topic for AI extraction

### Structured Data per Cluster
- Pillar: Article schema + FAQ schema
- Clusters: Article schema + HowTo/FAQ as appropriate
- All: BreadcrumbList schema showing hierarchy

## Workflow

### Monthly Authority Building
1. **Audit**: check cluster coverage gaps
2. **Create**: 2-4 cluster posts per month
3. **Link**: add internal links to existing posts
4. **Refresh**: update oldest cluster posts
5. **Measure**: track authority score trend

### Quarterly Review
1. Review all pillar + cluster content
2. Update outdated information
3. Add new cluster posts for emerging subtopics
4. Strengthen internal linking
5. Update authority score

## Pitfalls
1. Don't create clusters without pillar — hub must exist first
2. Don't skip internal linking — clusters must connect
3. Don't duplicate topics — each cluster unique
4. Don't ignore weak clusters — fill gaps before expanding
5. Don't forget to refresh — stale clusters lose authority

## Verification
- Pillar page published and linked to all clusters
- All clusters link back to pillar
- Internal links contextual and varied
- Cluster coverage measured (target > 80%)
- Authority score calculated and tracked
- Monthly growth trend positive
