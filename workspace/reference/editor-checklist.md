## 2. VOICE & TONE CHECK

### Yang Harus ADA:
- [ ] Sapaan "sobat" / "kamu" / "kalian" (BUKAN "Anda")
- [ ] Opening langsung ke value (sapa → konteks → masuk topik)
- [ ] Tone santai seperti ngobrol sama temen
- [ ] Sentuhan personal (pengalaman, opini, humor ringan)
- [ ] Bahasa Indonesia informal/semi-formal dengan istilah teknis English natural
- [ ] Kalimat pendek. Paragraf max 3-4 kalimat

### Yang TIDAK BOLEH ADA:
- [ ] Terlalu formal / kayak Wikipedia / kayak skripsi
- [ ] Tidak ada personality (bisa ditulis siapa aja)
- [ ] Wall of text tanpa break
- [ ] "Sobat" muncul lebih dari 3x per artikel (overuse = kaku)

### Opening Rules:
| ✅ BAGUS | ❌ JELEK |
|----------|----------|
| "Hello... sobat BagasUnix kasih tau cara..." | "Di era digital saat ini..." |
| "Malam sobat ^_^ Ketemu lagi..." | "Pada kesempatan kali ini..." |
| "Oke langsung saja..." | "Sebagai salah satu..." |
| "Kalau kamu pernah ngalamin..." | "Dalam dunia teknologi yang terus berkembang..." |
| Langsung masalah + personal context | Kalimat generik yang bisa di-copy ke blog mana aja |

## 4. AI SIGNS — Post-Edit Scan

**Wajib jalanin ini SETELAH edit selesai.** Scan HANYA baris yang diubah.

### 4.1 Meaning Escalation (Penggandaan Makna)
- "Ini menunjukkan/merepresentasikan/mencerminkan..." → facts speak for themselves
- "Di tengah lanskap yang terus berubah..." → scope inflation
- "Membuktikan bahwa..." → unless literally proven

### 4.2 Marketing Tone
- "Revolutionary", "groundbreaking", "cutting-edge", "seamless", "empowering"
- "Ternyata..." (clickbait framing di body text)

### 4.3 Sentence Patterns
- **"Not X, but Y"** — max 1 per artikel. Two = suspicious
- **Parallel triple** — "X, Y, dan Z" di mana Z dipaksa
- **Synonym swapping** — humans repeat naturally, AI avoids repetition
- **Short sentence barrage** — "No turning back. No Plan B." — 1 = OK, row = performance
- **Self-Q&A** — "Yang terjadi? Kacau." — artificial drama
- **Fake honesty** — "Jujur aja," diikuti point yang gak controversial

### 4.4 Formatting Traces
- **Em dash (—)** — paling reliable AI signal. Ganti semua:
  → period (split jadi 2 kalimat)
  → comma (compact)
  → colon (introduce explanation)
- **Bold abuse** — mechanical bolding keywords, "**Bold start:** explanation"
- **Emoji decoration** — 🚀💡✅ sebelum header/bullet
- **Title parrot** — first sentence setelah header = paragraf header dalam kata berbeda

### 4.5 Tone Tells
- "Hope this helps!" / "Semoga bermanfaat!" (di akhir, terlalu umum)
- "Here's the thing..." / "Yang menarik adalah..."
- "Experts say" / "Para ahli berpendapat" → sebut sumber atau hapus
- "Time will tell" / "Waktu yang akan membuktikan"
- "Let's dive in" / "Mari kita bahas"

### 4.6 Filler & Hedge
- "It's worth noting that..." / "Perlu dicatat bahwa..." → hapus, kalimat tetap jalan
- "It's important to mention..." / "Yang perlu ditegaskan..." → hapus
- "Mungkin barangkali" → stack of hedges
- "Dalam rangka" → "untuk"
- "In order to" → "to"

### Cluster Rule (Auto-Detect AI)
Em dash + triple parallel + "revolutionary" + "in conclusion" = guilty cluster.
Individual signal = probably fine, tapi kalau cluster muncul = rewrite section.

## 5. FACT-CHECK & ACCURACY

### Critical Checks:
- [ ] Semua klaim punya source (link atau citation)
- [ ] Versi software/tools masih current (cek PyPI/GitHub)
- [ ] Command/code bisa dijalankan (copy-paste test mental)
- [ ] Statistik/angka ada sumbernya
- [ ] Tanggal/tahun accurate
- [ ] Nama produk exact case (Docker, GitHub, VS Code — BUKAN docker, github, vscode)

### Red Flags (Likely Hallucination):
- Angka terlalu spesifik tanpa source
- "Menurut penelitian..." tanpa citation
- Feature "baru dirilis" — verify existence
- URL yang disebut — test apakah exist
- Framework/library yang belum ada di PyPI/GitHub

### Link Validation:
- [ ] Semua external link HTTP reachable (200-399)
- [ ] Tidak ada link ke domain hoax/fake-news
- [ ] Tidak ada redirect ke domain berbeda (suspicious)
- [ ] Minimal 1 sumber high-credibility (.edu, .gov, established media)
- [ ] Tidak ada dead link (404/410)

## 6. READABILITY

### Target Metrics:
- Flesch-Kincaid Grade Level: 8-10 (SMA level)
- Average sentence length: < 20 words
- Paragraph length: max 4 sentences
- Passive voice: < 10%

### Manual Check:
- [ ] Baca keras-keras. Kalau perlu baca ulang = too complex
- [ ] Split long sentences di "yang", "karena", "sehingga"
- [ ] Break wall-of-text jadi bullets
- [ ] Code blocks di tutorial harus lengkap + bisa di-copy-paste

---

## 7. STRUCTURE & SEO

### Structure:
- [ ] H2/H3 untuk sub-section (BUKAN H1 di dalam artikel)
- [ ] H2 scannable standalone (reader gets gist from headings)
- [ ] Logical flow (no jumping topics)
- [ ] Opening: sapa → value → masuk topik
- [ ] Closing: singkat + CTA (komentar/share) + BACA JUGA internal link
- [ ] Penutup: pendek, bukan essay panjang

### SEO:
- [ ] Primary keyword di title + first 100 words
- [ ] Internal links present (min 2-3, natural anchor text)
- [ ] External authority links (min 1-2)
- [ ] Slug: pendek, lowercase, keyword-focused
- [ ] Meta description: 1 kalimat yang bikin penasaran (bukan summary)

### Code Block Standards:
- [ ] Selalu pakai code block untuk command/config
- [ ] Language tag di code block (```bash, ```yaml, ```python, dll)
- [ ] Jelaskan KENAPA command dijalankan, bukan cuma copy-paste
- [ ] Warning untuk command berbahaya (rm -rf, format, dll)
- [ ] Output yang diharapkan ditunjukkan

---

## 8. FORMATTING STANDARDS

### Emoticon & Emoji:
- Emoticon: ^_^ :D :v — hemat, max 1-2x per artikel
- Emoji di body: minimal. Hindari 🚀💡✅ decoration
- Emoji di closing: boleh 1 (🐳 🎉 dll)

### Internal Link Section:
- Gunakan heading "BACA JUGA:" (caps)
- Anchor text natural, bukan generic "klik di sini"
- Minimal 2 link, relevan dengan topik

### Tables:
- Gunakan untuk perbandingan (Contoh: Docker Compose vs docker run)
- Format Markdown yang valid
- Jangan pakai table untuk konteks non-perbandingan

### Lists:
- Bullet untuk features/pros/cons
- Numbered untuk steps/langkah-langkah
- Max 7-8 items per list (split kalau lebih)

## 9. ARTICLE TYPE CHECKLISTS

### Tutorial:
- [ ] Hook/sapa 1-2 kalimat
- [ ] Kenapa penting (pengalaman/konteks)
- [ ] Prerequisites kalau ada
- [ ] Langkah bernomor, tiap langkah ada code block
- [ ] Troubleshooting / "kalau gagal..."
- [ ] Penutup singkat + ajakan komentar
- [ ] BACA JUGA section

### Listicle/Review:
- [ ] Hook yang relatable
- [ ] List dengan heading per item
- [ ] Tiap item: penjelasan + opini/pengalaman
- [ ] Verdict/kesimpulan yang jelas

### News:
- [ ] Headline langsung ke fakta
- [ ] Kronologi singkat
- [ ] Dampak + opini
- [ ] Penutup

## 10. FINAL GATE CRITERIA

Artikel **LOLOS** kalau semua ini terpenuhi:

### Must Pass (Blocking):
- [ ] ZERO "Generated by AI" / "Created by AI" text
- [ ] ZERO TODO/placeholder tertinggal
- [ ] ZERO blacklisted phrases (Section 3)
- [ ] Opening bukan AI-pattern (Section 2)
- [ ] Semua fact verified
- [ ] Semua link reachable
- [ ] Bahasa konsisten casual dari awal sampai akhir

### Should Pass (Strong Advisory):
- [ ] No em dash cluster (max 1 per 500 kata)
- [ ] No parallel triple pattern
- [ ] Paragraph max 4 kalimat
- [ ] Code blocks dengan language tag
- [ ] Internal link section ada
- [ ] Penutup singkat (bukan essay)
- [ ] Sentuhan personal ada

### Nice to Have:
- [ ] Emoticon 1-2x (hemat)
- [ ] Troubleshooting section (tutorial)
- [ ] Comparison table (review/listicle)
- [ ] Screenshot untuk UI tutorial

## 12. REVIEW WORKFLOW

1. **Scan red flags dulu** (Section 1) — auto-reject kalau ada
2. **Voice check** (Section 2) — pastikan casual + personal
3. **AI signs scan** (Section 4) — scan HANYA baris yang diubah
4. **Fact-check** (Section 5) — verify semua klaim + links
5. **Readability** (Section 6) — baca keras-keras
6. **Structure & SEO** (Section 7) — checklist
7. **Formatting** (Section 8) — code blocks, tables, lists
8. **Final gate** (Section 10) — semua "Must Pass" terpenuhi
9. **Run pipeline command**: `python3 scripts/pipeline.py review <idea_id>`
10. **Gate**: `python3 scripts/pipeline.py gate <idea_id>`
