## 1. Opening Pattern yang Works

### ✅ AUTHENTIC (era 2015-2019, recapture ini)
```
Sobat, kalau kamu googling "topic" sekarang, hasilnya kayak warung rame — banyak banget pilihan, masing-masing teriak-teriak "gue yang terbaik!"
```
```
Hello... sobat BagasUnix kasih tau cara mengatasi "masalah" pada device kamu, eh sebetulnya bukan kasih tau sh sekedar pengalaman saya aja yang ingin saya bagi kan.
```
```
Oke sobat, jadi kamu udah berhasil install X. Selamat. Tapi coba bayangin situasi ini: ...
```

### ❌ JANGAN PERNAH (AI tells yang sudah terdeteksi)
```
Di era modern ini, data menjadi sumber daya yang sangat berharga.
```
```
Bermain game menjadi salah satu aktivitas sebagian orang untuk melepas jenuh.
```
```
Platform ini hadir sebagai solusi yang memungkinkan Anda untuk...
```

### Formula Opening (3 langkah)
1. **Sapa langsung** (1 kalimat): "Sobat," / "Oke sobat,"
2. **Konteks relatable** (1-2 kalimat): situasi/frustrasi yang reader alami
3. **Janji value** (1 kalimat): "Di tutorial ini kamu bakal langsung praktek. Bukan cuma teori."

## 2. Panjang Paragraph Ideal

| Elemen | Panjang | Notes |
|--------|---------|-------|
| Paragraph | 2-3 kalimat max | Mobile-first, 85%+ reader di HP |
| Kalimat | 8-25 kata | Variasi: campur pendek (5 kata) + panjang (30 kata) |
| Section (H2) | 200-300 kata | Harus ada visual break (gambar/code block) |
| Artikel total | 1500-3000 kata | Tutorial: lebih panjang. Listicle: lebih pendek |

**Rule:** Kalau paragraph lebih dari 3 kalimat → SPLIT. Wall of text = reader kabur.

## 3. Gaya Penjelasan Teknis

### ✅ YANG BENAR
```bash
# Install Docker Engine dulu (kalau belum)
sudo apt update && sudo apt install docker.io

# Verifikasi — harus muncul versi
docker --version
```
- Jelaskan **KENAPA** command dijalankan, bukan cuma "jalankan ini"
- Kasih warning kalau command berbahaya (rm -rf, format, dll)
- Screenshot untuk UI-based tutorial
- Code block dengan comment inline

### ❌ YANG SALAH
- Cuma kasih command tanpa penjelasan
- Cuma copy-paste documentation tanpa opini
- Terlalu formal: "Perintah berikut digunakan untuk menginstal..."
- Tidak ada troubleshooting dari pengalaman pribadi

## 4. Yang Boleh dan Tidak Boleh

### ✅ BOLEH
- Pakai "sobat", "kamu", "kalian" sebagai sapaan
- Campur EN-ID natural: "install", "update", "deploy", "server"
- Humor ringan dan self-deprecating
- Emoticon secukupnya: ^_^ :D :v (max 1-2x per artikel)
- Personal anecdotes: "Gue nulis ini karena sempet frustasi sendiri"
- Opini kuat: "Semua orang bilang konsisten itu kunci. Salah."
- Frasa khas: "langsung aja", "semoga bermanfaat", "BACA JUGA"
- Frustration relatable: "keselnya notif tersebut muncul terus"

### ❌ TIDAK BOLEH
- "Anda" (terlalu formal) → pakai "kamu"/"sobat"
- "Di era digital/modern saat ini..."
- "Menjadi sumber daya yang sangat berharga"
- "Hadir sebagai solusi yang..."
- "Perlu diingat bahwa..."
- "Tidak dapat dipungkiri..."
- "Sangat penting untuk..."
- "Dalam rangka..."
- "Sebagai penutup..." / "Demikianlah..."
- "Dengan demikian..."
- "Memungkinkan Anda untuk..."
- Opening generik yang bisa dipasang di blog manapun
- Artikel tanpa pengalaman pribadi
- "Generated by AI" / "Created by AI" label

## 5. Code Block Format

### Yang benar
````markdown
```bash
# Comment menjelaskan kenapa
sudo apt update && sudo apt install package-name

# Verifikasi install
package --version
# Output: package x.x.x
```
````

### Pattern per step
1. **Judul step** (bold atau H3): "Install X"
2. **Kenapa** (1 kalimat): "Kafka butuh JDK buat jalan."
3. **Command** (code block dengan comment)
4. **Verifikasi** (code block: output yang diharapkan)
5. **Troubleshooting** (kalau error umum): "Kalau muncul error Y, ..."

### Syntax highlighting
- `bash` untuk shell commands
- `yaml` untuk config files
- `python` untuk Python code
- `json` untuk JSON output
- `text` untuk plain output/log

## 7. Internal Linking

- 2-4 internal link per artikel
- Pakai section **BACA JUGA** (bold) sebelum list link
- Anchor text deskriptif, bukan "klik di sini"
- Link ke related existing posts (cek feed blog)
- Jangan link ke post yang sama 2x dalam 1 artikel

## 8. SEO Quick Rules

- **Judul:** keyword utama + energi (! / angka)
- **Meta description:** ≤155 chars, 1 kalimat yang bikin penasaran
- **Slug:** pendek, lowercase, keyword-focused
- **H2/H3:** gunakan secondary keywords secara natural
- **Keyword density:** 1-2%, jangan stuffing
- **Label:** reuse existing labels blog (jangan bikin baru)

## 9. Quality Checklist (sebelum kirim ke editor)

- [ ] Opening BUKAN kalimat generik?
- [ ] Ada sentuhan personal (pengalaman, opini, humor)?
- [ ] Bahasa santai konsisten dari awal sampai akhir?
- [ ] Tidak ada paragraph >3 kalimat tanpa break?
- [ ] Kalau tutorial: ada troubleshooting dari pengalaman?
- [ ] Tidak ada kata/frasa dari daftar "TIDAK BOLEH"?
- [ ] 2-4 internal link ke post yang relevan?
- [ ] Code blocks dengan comment inline?
- [ ] Emoticon tidak lebih dari 2x per artikel?
- [ ] Bisa dibaca nyaman di HP (mobile-first)?

## 10. Evolution Notes

Blog punya 2 era voice:
- **2015-2019:** Voice asli — sangat personal, casual, ada humor dan emoticon
- **2020-2024:** AI-generated — formal, generik, tidak ada personal touch

**TARGET voice untuk artikel baru:** campuran — personality era 2015-2019 + struktur & depth era 2024.
