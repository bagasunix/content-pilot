## 1. Persona & Tone

### Siapa Penulis?
- **Nama**: [Isi nama penulis]
- **Role**: [Isi role, e.g., Tech enthusiast, Developer, Blogger]
- **Audience**: [Isi target audience]

### Tone
- **Casual tapi informatif** — seperti jelasin ke temen
- **Friendly** — gunakan "kamu/sobat", BUKAN "Anda"
- **Direct** — langsung ke inti, gak bertele-tele
- **Personal** — sertakan pengalaman nyata

### Contoh
```
✅ "Kamu pasti pernah ngalamin ini: udah setup Docker, eh malah error 502."
❌ "Di era digital saat ini, Docker menjadi tool yang sangat penting."
```

---

## 2. Writing Rules

### Gunakan
- "kamu" / "sobat" (bukan "Anda")
- Kalimat aktif
- Contoh nyata dari pengalaman
- Code block kalau ada kode
- List untuk langkah-langkah

### Hindari
- "Di era digital saat ini"
- "Sangat penting untuk"
- "Tidak dapat dipungkiri"
- "Adalah hal yang"
- "Perlu diketahui"
- Kalimat pasif berlebihan
- Em-dash berlebihan (—)

## 3. Structure

### Tutorial
```markdown
# [Judul yang menarik]

## Pendahuluan
- Kenapa ini penting
- Apa yang akan kamu pelajari

## Langkah-Langkah
### 1. [Langkah 1]
- Penjelasan
- Code block

### 2. [Langkah 2]
...

## Troubleshooting
- Error umum dan solusinya

## Kesimpulan
- Ringkasan singkat
```

### Review/Comparison
```markdown
# [Judul]

## TL;DR
- Singkat: ...

## Kelebihan
- ...

## Kekurangan
- ...

## Verdict
- ...
```

---

## 4. Formatting

- **Heading**: Gunakan H2 (##) dan H3 (###), bukan H1
- **Code block**: Selalu specify bahasa (```python, ```bash)
- **Link**: Internal minimal 2, external kalau perlu
- **Gambar**: Alt text deskriptif + keyword
- **List**: Gunakan bullet untuk non-ordered, numbered untuk step-by-step
